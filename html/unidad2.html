Unidad 2: Implementación de API

Material y ejercicios para la implementación de APIs, además de tecnologías y servicios asociados (geolocalización, redes sociales, bases de datos, despliegue y streaming).

Objetivos de la unidad:

  • Diseñar endpoints RESTful con buenas prácticas de versionado y seguridad.
  • Integrar servicios externos: geolocalización, redes sociales y plataformas en la nube.
  • Seleccionar y usar la base de datos adecuada según el caso.
  • Preparar la aplicación para despliegue en plataformas modernas y consumo de streaming.

Geolocalización — conceptos

Uso de APIs y servicios para obtener coordenadas, resolver direcciones y ofrecer servicios basados en la ubicación del usuario o recursos.

  • Fuentes: Geolocation API (navegador), Google Maps Platform, OpenStreetMap/Nominatim.
  • Casos de uso: búsqueda local de servicios, ruteo, personalización por ubicación.
  • Consideraciones: permisos, precisión, consumo de cuotas y uso de HTTPS.

Ejemplo: Geolocalización (JS)

if (navigator.geolocation) {
  navigator.geolocation.getCurrentPosition(
    function(pos) {
      console.log('Lat:', pos.coords.latitude);
      console.log('Lng:', pos.coords.longitude);
    }, 
    function(err) { 
      console.error(err.message); 
    }, 
    { enableHighAccuracy: true, timeout: 10000 }
  );
} else {
  console.error('Geolocation not supported');
}

Redes Sociales — integración

Conectar con APIs sociales para autenticación, publicación automática y monitorización de interacciones.

  • Mecanismo estándar: OAuth 2.0 para autorización segura.
  • Webhooks: recibir eventos en tiempo real (mensajes, publicaciones).
  • Manejo de límites: caching, backoff y reintentos controlados.

Ejemplo: Publicar (pseudo)

// POST /api/social/publish
fetch('/api/social/publish', {
  method: 'POST',
  headers: { 
    'Content-Type': 'application/json' 
  },
  body: JSON.stringify({ 
    platform: 'twitter', 
    text: 'Hola desde mi API' 
  })
})
.then(r => r.json())
.then(console.log)
.catch(console.error);

Bases de Datos — selección

Elegir entre relacionales y NoSQL según necesidades de consistencia, consultas y escalabilidad.

  • Relacionales (Postgres, MySQL): integridad, transacciones y consultas complejas.
  • NoSQL (MongoDB, Redis): documentos, caches, alta disponibilidad y escalado horizontal.

Ejemplo: Tabla usuarios (Postgres)

CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  username VARCHAR(50) UNIQUE NOT NULL,
  email VARCHAR(150) UNIQUE NOT NULL,
  created_at TIMESTAMP DEFAULT now()
);

Plataformas On-Line — despliegue

Servicios para despliegue rápido y escalado: Heroku, Vercel, Netlify, y proveedores cloud (AWS/Azure/GCP).

  • Elección basada en control requerido, coste y facilidad de CI/CD.
  • Serverless vs contenedores vs IaaS — decidir según latencia y estado.

Ejemplo: Despliegue rápido (Vercel)

npm i -g vercel
vercel login
vercel --prod

Plataformas Streaming — conceptos

Protocolos y desafíos para entrega de video/audio en vivo y bajo demanda.

  • RTMP: para ingestión de video.
  • HLS/DASH: para reproducción adaptativa.
  • Consideraciones: CDN, codificación, latencia, DRM y costes de ancho de banda.

Buenas prácticas de integración

  • Documentar APIs con OpenAPI/Swagger y mantener ejemplos actualizados.
  • Asegurar endpoints con HTTPS, autenticación y autorización (OAuth2, JWT).
  • Implementar pruebas: unitarias, integración y end-to-end (Postman, Newman).
  • Monitoreo y logs centralizados (ELK, Prometheus, Grafana).

Recursos recomendados:

  • Swagger / OpenAPI
  • MDN Web Docs
  • Google Maps Platform
